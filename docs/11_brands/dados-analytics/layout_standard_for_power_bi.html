01

Objetivo do Dashboard

Este documento estabelece um padrão estrutural único para construção de páginas no Power BI. O objetivo não é restringir análise, e sim padronizar a forma como ela é apresentada.

Consistência
Visual uniforme entre páginas e relatórios
Previsibilidade
Layout estável para manutenção e evolução
Reutilização
Facilidade de template via .pbit
02

Formatos de Página

Paisagem Recomendado
2040 × 1000 px · Altura fixa
  • Visão geral e páginas executivas
  • Comparações de alto nível
  • Leitura horizontal
Cada página deve ter um propósito claro e único.
Retrato Análise detalhada
2040 px · Altura variável
  • Análises detalhadas e páginas longas
  • Exploração progressiva do conteúdo
  • Organização vertical dos blocos
Alinhamento lateral e espaçamentos são obrigatórios mesmo com altura variável.

Quando houver
📊
KPIs funcionam como elementos de leitura rápida. Devem estar organizados em grid, com alinhamento horizontal e espaçamento uniforme. Nunca posicionados isoladamente no meio da página. KPIs não substituem outros elementos analíticos — eles apenas abrem a leitura.
07

Consistência entre Páginas

O usuário não deve precisar reaprender o layout ao navegar entre páginas.

Margens externas de 32 px em todas as páginas
Espaçamento entre blocos de 16 px em todas as páginas
Header fixo de 94 px em todas as páginas
Mesma hierarquia estrutural em todas as páginas
08

Governança de Layout

O que pode variar
  • Quantidade de blocos no corpo
  • Altura da página (formato retrato)
  • Combinação de componentes internos
O que não pode variar
  • Margens
  • Espaçamentos
  • Header
  • Posição do título
  • Estrutura geral da página
🏗️
Liberdade analítica existe dentro da estrutura, não fora dela.
09

O que Não é Permitido

✕ Quebra de margens ou espaçamentos
✕ Header diferente entre páginas
✕ Elementos fora do grid
✕ KPIs fora do topo da página
✕ Ajustes manuais "para caber melhor"
⚠️
Exceções exigem justificativa clara e documentada.
📐
Estrutura é infraestrutura
O padrão de layout garante que qualquer pessoa da equipe possa manter, evoluir ou criar um dashboard sem recomeçar do zero. Desvios do padrão geram retrabalho, inconsistência e ruído visual para o usuário final.
